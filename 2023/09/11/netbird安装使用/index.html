netbird安装使用

发表于 2023-09-11 更新于 2023-10-10 阅读次数： 本文字数： 7.4k 阅读时长 ≈ 7 分钟

对我来说netbird比headscale好处就是可以分组自定义peer路由，完全实现服务端控制下发路由，且下发和回收路由速度几乎秒级(headscale关闭路由后很长时间不清理路由)

安装

使用oth0方式

官方文档: https://docs.netbird.io/selfhosted/selfhosted-guide#advanced-guid

  1. 参照文档服务器放行端口

  2. 准备一个域名 netbird.example.com

  3. clone release 代码

  4. 已Auth0作为认证

    1. https://auth0.com注册账号

    2. 创建NetBird Login应用

    3. Application Login URI 填入 https://netbird.example.com

    4. Allowed Callback URIs、Allowed Logout URLs 与 Allowed Web Origins 都填入 https://netbird.example.com,http://netbird.example.com

    5. 启用Device Code授权方式：

    6. 创建和配置机器对机器应用程序。

      此应用程序将用于授权对Auth0 Management API的访问。

      勾选权限

      配置：

      • Copy Client ID to NETBIRD_IDP_MGMT_CLIENT_ID in the setup.env file
      • Copy Client SECRET to NETBIRD_IDP_MGMT_CLIENT_SECRET in the setup.env file
      • Copy DOMAIN to NETBIRD_IDP_MGMT_EXTRA_AUDIENCE in the setup.env file

    7. 创建和配置Auth0 API

      此Auth0 API将用于访问NetBird管理服务API。

      https://auth0.com/docs/quickstart/backend/golang#create-an-api

  5. 执行初始化

  6. 访问域名后注册用户

使用自建OIDC(keycloak)

部署keycloak

  1. 参照官方文档：https://docs.netbird.io/selfhosted/identity-providers#keycloak

  2. 流程结束后访问https://< YOUR_KEYCLOAK_HOST_AND_PORT >/realms/netbird/.well-known/openid-configuration 有数据就ok

  3. 重新部署时记得把docker的所有卷及数据清理干净

  4. 使用netbird残留数据搭建会产生权限异常问题,如果遇到权限问题，即页面能打开，但没有管理界面

    ​ 将Role 修改 为 admin

使用

修改默认的IP地址池(默认的地址池会和阿里云内网的服务冲突，导致无法访问，比如使用aliyunlog下载日志)

在Setup Keys页面中创建key并复制
